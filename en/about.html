About·PasswordZ

Original password + scene → a strong password. Open algorithm, never lost; computed locally, never leaked.

Problem

What problem does PasswordZ solve?

Have you ever had any of these concerns?

  • Too many passwords to remember? How do you keep track of passwords for so many websites, apps, and bank cards?
  • One password everywhere? What happens when one site's password is leaked? You'd have to change it across every website, app, and bank card!
  • Different password rules? Site A requires letters and digits, site B forbids special characters, site C allows only 8 digits. Forgetting the rules means endless resets — and then forgetting again!
  • Password managers? What if you don't have your computer, or you're borrowing someone else's device and can't use a password manager?

PasswordZ's solution

One original password, infinitely many independent distribution passwords

Generate different distribution passwords from one original password combined with different scenes. Remember just one original password, and with PasswordZ every website, app, and bank card gets an independent, unrelated password.

One-way, irreversible generation algorithm

The generation algorithm is irreversible, with the scene code and character set layered in as encryption factors. Even if one site's password leaks, the original password cannot be reverse-engineered from it.

High-strength passwords

16-character combinations of digits, upper/lowercase letters, and special characters significantly increase password strength.

Adapts to each site's password rules

Customize the character set to flexibly generate passwords that meet both your security needs and each site's rules.

Highly available, anywhere

Offered as OS distributions, browser extensions, and an online service — at least two options on any device, so you can use it anytime, anywhere.

Security

Is it safe to entrust your passwords to PasswordZ?

Traceless — stateless, sessionless, no storage, no leaks

PasswordZ is stateless and sessionless, with no server-side or local storage. Using it leaves no trace locally or on the network, eliminating password leaks at the source.

Local computation — nothing to steal

PasswordZ computes entirely locally (the online version computes within the browser). During computation it touches no local disk and no network, so there is no way for a third party to intercept it.

Open-source algorithm — never lost

The PasswordZ algorithm is open source, and its underlying crypto is open source too. With online, offline, and browser-extension builds officially provided, there is no scenario where the algorithm becomes unusable or lost.
